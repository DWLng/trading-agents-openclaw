## 投资风格详解（Agent决策参考）

### 风险偏好：高风险偏好 + 纯右侧交易

**含义**：用户是严格的右侧趋势交易者+打板交易者。宁愿追高也不抄底，宁愿不做也不错做。追求趋势确认后的主升浪，不接受左侧埋伏和横盘等待。

**三条铁律（硬性约束）**：
1. 不做左侧：股价处于下降趋势/新低/阴跌 = 直接弃牌
2. 不做横盘：30日内振幅<15%且无方向 = 不浪费时间
3. 不抄底：趋势确认前，任何PE/基本面优势都忽略

**Agent适配**：
- 评分体系必须反映趋势强度：技术面35% + 资金面25% + 趋势动量15% + 概念12% + 消息8% + 基本面5%
- <60分 = 不看/不买/不分析，坚决弃牌
- 重点识别：突破历史新高、放量涨停、最强主线板块
- 止损线-8%，但趋势破坏就无条件走

### 时间偏好：超短线（1-3天，最多1周）

**含义**：打板为主，持仓超过3天属于异常，超过1周必须认真审视是否该走。

**Agent适配**：
- 重点关注"次日溢价"而非"长期价值"
- 技术指标以日线/分时为主，不看周线月线
- 催化剂必须是"近3天内可兑现"的事件
- 涨停次日+3-5%先出一半

### 操作风格：追涨打板 + 趋势跟随

**含义**：只买已经涨的。涨停封板=确认信号。

**Agent适配**：
- 选股时"趋势动量"权重15%，涨停基因是核心指标
- 推荐时机必须是突破确认后/涨停当日/次日竞价
- 封板质量（封单量/开板次数）是关键决策信息
- 板块共振 > 个股独立行情

### 容忍回撤：10-15%

**含义**：用户不会因为10-15%的正常波动而恐慌。

**Agent适配**：
- 盘中监控的预警阈值设为-10%（提醒）/-15%（建议止损）
- 推荐时主动说明"持仓期间可能出现X%的波动，确认能接受吗"
- 不因短期回撤（<10%）而频繁调整策略参数

### 信息偏好：政策驱动 + 资金流向 + 技术面

**含义**：用户重视宏观政策和资金面的合力。

**Agent适配**：
- 在消息面分析中，政策信号的权重高于公司公告
- 资金面分析（北向、主力、游资）与技术信号并重
- 板块分析时关注"政策催化 + 资金流入"的共振机会

## 风格冲突解决（当市场状态与用户风格冲突时）

| 市场状态 | 用户风格 | 冲突 | Agent处理 |
|---------|---------|------|----------|
| 市场极度低迷（无量阴跌） | 高风险偏好，要波动 | 用户要波动，但市场没波动 | 坦诚报告"当前市场缺乏明确趋势，建议降低仓位或观望"，不强行推荐 |
| 市场暴涨（全面牛市） | 中短线，1-4周 | 牛市中短线容易卖飞 | 建议分批止盈策略，"锁定部分利润，让利润奔跑" |
| 用户持仓股暴跌（超预期） | 容忍15%回撤 | 回撤可能超过容忍度 | 立即预警，区分"正常波动"和"趋势破坏"，如果是后者，建议止损 |
| 用户要求分析防御性板块 | 重赔率，怕横盘 | 防御性板块通常低波动 | 如实说明"该板块赔率一般，但当前市场环境下的配置价值"，让用户自选 |

## 隐性需求推断

从用户历史行为推断可能的需求（基于memory中的交互记录）：

- 如果用户连续3次询问同一只股票 → 推断为"重点关注"，增加该股的主动监控频率
- 如果用户在某板块盈利后频繁询问该板块 → 推断为"该板块正反馈"，适度增加该板块在选股中的权重
- 如果用户在亏损后询问"市场怎么样"而非具体股票 → 推断为"情绪需要稳定"，先给市场定位，再给机会
- 如果用户多次提到"忙""没空看" → 推断为"需要精简推送"，减少盘中通知，增加收盘汇总

## 反馈循环

用户很少直接说"我喜欢/不喜欢这个风格"，Agent需要从以下信号推断：
- 对某类推荐的后续追问频率（追问 = 兴趣）
- 是否按推荐执行了模拟买入（执行 = 认可）
- 对飞书文档的打开频率（如技术允许追踪）
- 亏损后的反应（继续信任 vs 质疑风格）

每两周做一次"用户画像校准"，将推断结果以摘要形式写入 `memory/user-profile-updates.md`，并在下次交互时微调风格。
