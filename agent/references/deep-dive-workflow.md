# 上市公司超级深度分析框架（13维融合版）

> **版本**: V2.0 Fusion | **融合来源**: 原12维深度分析框架 + 九维深度拆解工作流 | **适用对象**: 具备完整财报的A股/港股/美股上市公司
> **设计哲学**: 九维框架提供"业务-产业链-基本面"的深度拆解能力，12维框架提供"技术面-资金面-消息面-宏观"的市场维度视角。融合后的13维框架专为具备完整财报的上市公司设计，实现"从生意本质到市场博弈"的全景覆盖。

---

## 一、框架概述

### 1.1 融合逻辑说明

本框架是原12维深度分析与九维深度拆解的融合升级版。融合逻辑如下：

| 来源框架 | 保留的独有维度 | 融合的重叠维度 |
|----------|---------------|---------------|
| **九维框架** | 公司画像、历史与发展、业务拆解（BOM/产业链/客户）、成长天花板、地缘政治灰犀牛 | 行业分析（规模/集中度+景气度/政策）、竞争格局（对标分析）、护城河（四维评估）、财务透视（7年趋势）、股东筹码 |
| **12维框架** | 技术面深度分析、资金面深度分析、催化剂日历、宏观环境映射 | 行业分析（景气度/政策面）、竞品对比、财务健康度、股东变化追踪 |

**融合后13维度**：公司画像 → 历史与发展 → 业务拆解 → 产业链与行业分析 → 竞争对标 → 护城河评估 → 财务深度透视 → 股东与筹码 → 成长天花板 → 风险评估 → 技术面深度分析 → 资金面深度分析 → 催化剂日历与宏观映射

### 1.2 适用边界

| 适用类型 | 说明 |
|----------|------|
| A股/港股/美股上市公司 | 具备完整定期报告（年报/季报）、公告披露 |
| 制造业/科技业企业 | 拥有可拆解的产品矩阵、BOM结构、技术路线 |
| 产业链中上游企业 | 涉及供应商/客户关系、议价权分析 |
| 快速成长/转型期企业 | 需要识别"关键一跃"、第二增长曲线 |
| 全球化运营企业 | 涉及地缘政治风险、跨国供应链布局 |

### 1.3 双引擎数据来源

本框架采用**mx-skills + wencai-skills**双数据引擎：

- **基本面/财务/产业链数据**: mx-skills API（公司财务、行业数据、公告、研报）+ wencai-skills（财务指标筛选、产业链图谱）
- **技术面/资金面数据**: mx-skills（K线、均线、成交量、资金流向、龙虎榜）+ wencai-skills（技术指标筛选、资金异动）
- **宏观/政策/消息面**: wencai-skills（政策解读、新闻舆情、行业景气度）+ mx-skills（宏观经济指标）
- **交叉验证**: 双引擎数据相互校验，不一致时标注⚠️数据分歧

## 二、13维深度分析框架

---

### 维度1：公司画像（Company Identity Card）

**目的**: 在30秒内建立对目标公司的全貌认知，快速定位其资本市场坐标与生意本质。

**分析内容**:

| 信息项 | 内容 | 数据来源 |
|--------|------|----------|
| 公司全称/成立时间/注册地 | 工商注册信息 | 天眼查/企查查/wencai |
| 上市时间/上市地/股票代码 | 交易所公开信息 | mx-skills/wencai |
| 控股股东/实际控制人 | 股权结构 | 年报/wencai |
| 注册资本/组织形式/审计机构 | 公司治理信息 | 年报 |
| 所属行业 | 申万一级→二级→三级穿透 | wencai行业分类 |
| 公司战略愿景 | 官方战略表述 | 年报/官网/业绩会 |
| 系族关系 | 母子公司/品牌架构 | 天眼查/集团官网 |

**输出格式**:
```
一级行业：[名称]（共XXX家上市公司）
  └── 二级行业：[名称]（共XXX家）
        └── 三级行业：[名称]（共XXX家）← 核心竞争圈

一句话生意描述：[公司]是[做什么]的[行业定位]，通过[核心商业模式]赚取[利润来源]。
```

### 维度2：历史与发展（Historical Milestones）

**目的**: 通过关键发展里程碑理解公司成长路径、战略选择与基因特质，识别"关键一跃"转折点。

**分析内容**:

| 时间 | 事件 | 战略意义 |
|------|------|----------|
| YYYY年 | 创始/创业起点 | 基因溯源 |
| YYYY年 | 技术突破/产品发布 | 能力跃迁 |
| YYYY年 | 上市里程碑 | 资本化节点 |
| YYYY年 | 重要并购/转型 | 战略转向 |
| YYYY年 | 全球化布局关键节点 | 空间扩张 |
| YYYY年 | 近期重大战略调整 | 当前方向 |

**关键一跃识别**: 找出公司从"普通"到"卓越"的转折点——通常是技术突破、大客户突破或产能跃迁的交汇点。

**管理层画像**: 创始人履历、企业家精神特质（轶事/第三方视角）、管理风格与公司治理特征。

### 维度3：业务拆解（Business Decomposition）

**目的**: 全面理解公司的业务结构、盈利模式和核心竞争力来源，回答"这家公司到底靠什么赚钱"。

**分析内容**:

**3.1 业务板块矩阵**

| 业务板块 | 营收占比 | 毛利率 | 同比增速 | 核心商业模式 |
|----------|----------|--------|----------|-------------|
| 板块1（主业务） | XX% | XX% | XX% | 一句话描述 |
| 板块2 | XX% | XX% | XX% | 一句话描述 |
| 板块3 | XX% | XX% | XX% | 一句话描述 |

分析要点：识别营收主力vs盈利主力（未必一致）、"隐藏盈利亮点"（营收占比小但毛利率高的业务）、板块间协同效应。

**3.2 核心产品矩阵**

| 产品/服务 | 生命周期 | 技术原理 | 定价模式 | 价值传递路径 |
|-----------|----------|----------|----------|-------------|
| 产品1 | 导入/成长/成熟/衰退 | 核心技术 | 成本+/价值/竞价 | 如何到达终端 |

**3.3 大客户绑定关系**

| 客户名称 | 关联度 | 收入贡献 | 合作深度 | 风险评级 |
|----------|--------|----------|----------|----------|
| 客户1 | 高/中/低 | XX% | 战略/认证/普通 | 高/中/低 |

分析要点：客户集中度风险、客户认证壁垒（周期/替代难度）、客户承诺兑现率追踪。

**3.4 BOM成本结构（制造业适用）**

| 成本项目 | 占比 | 趋势 | 驱动因素 |
|----------|------|------|----------|
| 原材料A | XX% | 升/降/稳 | |
| 原材料B | XX% | 升/降/稳 | |
| 人工 | XX% | 升/降/稳 | |

### 维度6：护城河评估（Moat Assessment）

**目的**: 识别并评估公司核心竞争壁垒的宽度和可持续性，判断优势是否可长期维持。

**分析内容**:

**6.1 护城河五维评估矩阵**（九维四维 + 12维专利壁垒融合）

| 护城河类型 | 具体表现 | 强度评级 | 可持续性 | 被侵蚀风险 |
|------------|----------|----------|----------|------------|
| **技术壁垒** | 专利数量、研发投入占比、技术领先代差 | ★★★☆☆ | 高/中/低 | |
| **专利壁垒** | 发明专利占比、核心专利被引用次数、专利诉讼历史 | ★★★☆☆ | 高/中/低 | |
| **客户认证壁垒** | 认证周期、客户粘性、转换成本 | ★★★★☆ | 高/中/低 | |
| **产能壁垒** | 资本密集度、建设周期、良率爬坡难度 | ★★★☆☆ | 高/中/低 | |
| **成本优势** | 规模效应、工艺know-how、良率优势 | ★★★★☆ | 高/中/低 | |
| **品牌/渠道** | 品牌溢价、渠道覆盖、售后服务网络 | ★★☆☆☆ | 高/中/低 | |

**6.2 综合护城河评级**

| 评级 | 定义 | 特征 |
|------|------|------|
| **宽阔且持久** | 多重壁垒叠加，转换成本高 | 技术+客户认证双壁垒，或垄断性资源 |
| **中等宽度** | 单一壁垒主导，存在被突破可能 | 依赖技术或成本之一，需持续投入维持 |
| **脆弱** | 主要靠价格竞争 | 壁垒容易被复制/削弱，差异化程度低 |

## 四、4级深度标准（Level 1-4）

本框架定义4个分析深度级别，根据时间预算和数据完整度自适应选择。

### Level 1：基础画像（Basic）

| 陷阱 | 表现 | 规避方法 |
|------|------|----------|

## 附录C：K线图快速预览URL方案（东方财富接口）

利用东方财富公开接口，无需数据获取和绘图代码，一行拼接生成带MACD的K线预览图。

### 适用场景

- 选股初筛阶段快速预览多只股票走势
- 盘中监控时快速查看异动股K线
- 表格/列表悬浮预览（PySide6/PyQt cellEntered事件）
- 用户只需看图不想等待matplotlib完整渲染

### 核心函数

```python
def get_stock_pic_url(code_str: str) -> str:
    """
    根据股票代码生成东方财富K线图（含MACD）的URL链接
    :param code_str: 股票代码，支持 '600000' 或 '600000.SH' 格式
    :return: 图片URL字符串，输入无效时返回空字符串
    """
    import pandas as pd
    if not code_str or pd.isna(code_str):
        return ""
    code_str = str(code_str).strip()
    if not code_str:
        return ""

    prefix = "0."
    if code_str.endswith('.SH'):       # 上海证券交易所
        prefix = "1."
        code_str = code_str[:-3]
    elif code_str.endswith('.SZ'):     # 深圳证券交易所
        prefix = "0."
        code_str = code_str[:-3]
    elif code_str.endswith('.BJ'):     # 北京证券交易所
        prefix = "0."
        code_str = code_str[:-3]
    else:
        # 智能推断：沪市6/5开头，深市0/3开头
        if code_str.startswith(('6', '5')):
            prefix = "1."

    url = (f"https://webquoteklinepic.eastmoney.com/GetPic.aspx?"
           f"nid={prefix}{code_str}&type=&unitWidth=-6&ef=&formula=MACD&AT=1&imageType=KXL")
    return url
```

### URL参数说明

| 参数 | 含义 | 可选值 |
|------|------|--------|
| `nid` | 市场前缀+代码 | `1.600000`（沪）、`0.000001`（深） |
| `formula` | 技术指标 | `MACD`（默认）、`KDJ`、空 |
| `imageType` | 图表类型 | `KXL`（K线图） |
| `unitWidth` | K线宽度 | `-6`（自适应） |

### 示例

```python
print(get_stock_pic_url("600000.SH"))  # 浦发银行
print(get_stock_pic_url("000001"))     # 平安银行
print(get_stock_pic_url("688981"))     # 科创板股票
```

### 注意事项

- 需联网调用，图片由东方财富服务器实时渲染
- 深色背景图片，不支持自定义配色
- 仅供快速预览，正式报告仍使用matplotlib/mplfinance生成高质量图表

### 10.0 关键原则

**HTML报告是必须交付物**，必须生成。

> ⚠️ **HTML报告是唯一交付物**，Cloudflare Pages部署失败时直接发送HTML文件。

### 10.1 报告生成架构

本框架支持生成专业HTML报告，架构如下：

```
Agent输出 Markdown/JSON
        ↓
report-generator.py 渲染
        ↓
完美HTML报告
        ↓
cf_pages_deploy.py 部署到 Cloudflare Pages
        ↓
分享链接给用户
```

### 10.2 独立执行代码

```python
# 1. 准备数据
report_data = {
    "title": "{股票名称}({代码}) 深度分析",
    "subtitle": "分析日期：YYYY-MM-DD | 分析深度：Level X",
    "content_markdown": markdown_content,
    "dashboard": [
        {"label": "综合评分", "value": "XX分", "icon": "star", "value_class": "text-blue-400"},
        {"label": "目标价", "value": "XX元", "icon": "target", "value_class": "text-green-400"},
        {"label": "止损位", "value": "XX元", "icon": "shield", "value_class": "text-red-400"}
    ],
    "score": XX
}

# 2. 写入临时文件
import json
with open("/tmp/report_input.json", "w", encoding="utf-8") as f:
    json.dump(report_data, f, ensure_ascii=False, indent=2)

# 3. 渲染HTML（使用绝对路径）
subprocess.run([
    "python3",
    "/Users/mac/.openclaw/agents/trading/skills/report-generator/report_generator.py",
    "/tmp/report_input.json"
])

# 4. 部署到Cloudflare Pages
subprocess.run([
    "python3",
    "/Users/mac/.openclaw/agents/trading/skills/cf-upload/cf_pages_deploy.py",
    "<上面生成的HTML文件路径>"
])
```

**Step 4: 返回链接给用户**
```
📊 深度分析报告已生成

贵州茅台(600519) 2026-05-10

🔗 网页版研报: https://xxx.trading-reports.pages.dev/deep-research_600519_20260510.html
```

### 10.3 HTML报告特性

| 特性 | 说明 |
|------|------|
| Tailwind CSS v4 | CDN引入，自动暗黑模式，响应式布局 |
| ECharts 5 | 支持K线图、资金流图动态交互 |
| 左侧导航栏 | 自动生成TOC，点击平滑滚动 |
| Dashboard卡片 | 顶部4格关键指标卡片 |
| 条件渲染 | 高分警告、风控Banner |
| 东方财富K线图 | 直接嵌入实时行情URL |

### 10.4 报告命名规则

> **命名规则**：纯ASCII格式（字母+数字+连字符），确保URL中可完整识别为超链接

| 类型 | 文件命名 | 说明 |
|------|---------|------|
| 选股日报 | `stock-pick_{YYYYMMDD}.html` | 每日收盘选股报告 |
| 个股深度分析 | `deep-research_{6位代码}_{YYYYMMDD}.html` | 13维度深度分析 |
| 个股快速分析 | `quick-analysis_{6位代码}_{YYYYMMDD}.html` | 六维快速分析 |
| 复盘报告 | `recap_{YYYYMMDD}.html` | 每日复盘报告 |
| 预警通知 | `alert_{6位代码}_{YYYYMMDD}.html` | 持仓预警报告 |
