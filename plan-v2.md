# Harness思维改造计划

## 目标
运用"Harness（缰绳）思维"改造OpenClaw Trading Agent配置：约束恰到好处——保住下限的同时开拓上限。

## 当前架构评估

**优势：**
- 数据源三层缓存（本地Parquet → mx-skills → AkShare）
- 主动（收盘选股/复盘）+ 被动（个股分析）双模式
- 飞书文档输出体系完整
- 记忆文件与策略进化闭环
- 风险红线清晰

**问题：**
1. 流程过于刚性（AGENTS.md Step 1→7），LLM无调整空间
2. 约束以负面清单为主，缺少正向引导
3. 无"元认知层"——何时该打破流程、何时该严守
4. 失败恢复机制薄弱
5. 核心价值观平铺，无冲突解决机制
6. 缺少"工作记忆"vs"长期记忆"区分

## Harness思维核心原则

1. **三层约束架构**：硬边界（不可逾越）→ 软引导（默认路径）→ 自由空间（创新区）
2. **流程模板化而非流程固化**：提供"默认舞步"，但允许根据情境即兴
3. **约束即释放**：清晰的边界内，赋予完全自主权
4. **元认知层**：让Agent能判断"我现在该守规矩还是该打破规矩"
5. **失败即进化**：每次异常都是更新认知模型的机会

- 改造全部8个文件

**改造后的文件清单：**
1. IDENTITY.md → 增加"情境身份切换"机制
2. SOUL.md → 重构为三层：核心真理 + 决策框架 + 弹性空间
3. BOOTSTRAP.md → 增加"首次对话诊断"和"能力导航"
4. AGENTS.md → 模板化流程 + 元流程控制 + 分支决策树
5. TOOLS.md → 工具组合自由度 + 失效创造性替代
6. HEARTBEAT.md → 动态优先级重排 + 弹性触发条件
7. USER.md → 投资风格参数化 + 适配矩阵
8. skills-lock.json → 保留不变（技术锁定文件）
9. METACOGNITION.md → **新增** 元认知：模式识别、冲突仲裁、创新触发
10. FAILURE_PROTOCOL.md → **新增** 失败分级、恢复策略、知识萃取
11. ADAPTIVE_CONSTRAINTS.md → **新增** 约束强度动态调节器
